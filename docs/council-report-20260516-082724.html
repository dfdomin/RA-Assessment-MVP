LLM Council — Lineas Propedeuticas y Resumen Ejecutivo

Fecha: 2026-05-16

Veredicto

No esta contemplado adecuadamente. El producto y el codigo estan modelados para assessment de un programa, no para una plataforma institucional con jerarquia Modulo -> Programa -> Linea Propedeutica.

Para soportarlo se requiere cambio mayor de dominio, permisos y reporting: entidades de programa/linea/ciclo, rol Decano o autoridad equivalente, y resumen ejecutivo agregado por linea.

Acuerdos del Consejo

Falta modelo de dominio
No existen Program, PropedeuticLine ni AcademicCycle.
Falta rol ejecutivo
Admin no equivale semanticamente a Decano.
Falta reporte agregado
Los reportes actuales son por periodo/programa, no por linea.
Es cambio de alcance
No conviene improvisarlo sobre Module/Period.

Primer Paso

Actualizar el PRD como cambio de alcance mayor: pasar de "v1 solo TGA" a plataforma institucional multi-programa, o registrar esta capacidad como sprint futuro independiente.

Detalles

Contrarian

Sin jerarquia persistente, cualquier resumen ejecutivo seria una convencion manual no auditable.

First Principles

La pregunta ejecutiva no puede responderse si los datos no conocen programa ni linea.

Expansionist

Esto abre una plataforma institucional con comparativos, tendencias, riesgos y drill-down.

Outsider

Si el sistema habla de lineas y decanos, esos conceptos deben existir en datos, permisos y pantallas.

Executor

Agregar entidades, migrar TGA al nuevo modelo, sumar rol ejecutivo, endpoints, docs y tests.
